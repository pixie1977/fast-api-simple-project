
# 🩺 Diabetes Prediction API

Сервис для предсказания риска диабета у пациента на основе антропометрических и биохимических данных.  
Реализован на FastAPI, использует ONNX-модель для инференса и защищён JWT-аутентификацией с поддержкой ролей.

---

## 🔧 Технологии

- **FastAPI** — для создания JSON API
- **ONNX Runtime** — выполнение модели машинного обучения
- **JWT (JSON Web Tokens)** — аутентификация и авторизация
- **RBAC (Role-Based Access Control)** — управление доступом по ролям
- **Pydantic** — валидация входных данных
- **Docker** — контейнеризация
- **Логирование** — отслеживание запросов

## 🔐 Аутентификация

Система использует JWT-аутентификацию:

1. Пользователь регистрируется через `/auth/register`
2. Аутентифицируется через `/auth/login` и получает JWT токен
3. Использует токен в заголовке `Authorization: Bearer <token>` для доступа к защищённым эндпоинтам

Поддерживается RBAC с ролями:
- **user** — может делать предсказания
- **admin** — имеет доступ к административным функциям

## 📥 Эндпоинт: `POST /auth/register`

Регистрирует нового пользователя.

### Входные данные (JSON)

| Поле         | Тип    | Ограничения | Описание |
|--------------|--------|-------------|---------|
| `username`   | `str`  | min_length=3 | Имя пользователя |
| `email`      | `str`  | валидный email | Email |
| `password`   | `str`  | min_length=6 | Пароль |

### Пример запроса

```bash
curl -X POST "http://127.0.0.1:8000/auth/register" \
  -H "Content-Type: application/json" \
  -d '{"username":"newuser", "email":"user@example.com", "password":"password123"}'
```

### Ответ (успешный)

```json
{
  "username": "newuser",
  "email": "user@example.com",
  "role": "user"
}
```

### Ошибки

| Код | Причина |
|-----|--------|
| `409` | Пользователь с таким именем уже существует |

---

## 📥 Эндпоинт: `POST /auth/login`

Аутентифицирует пользователя и возвращает JWT токен.

### Входные данные (form-data)

- `username` — имя пользователя
- `password` — пароль

### Пример запроса

```bash
curl -X POST "http://127.0.0.1:8000/auth/login" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d 'username=admin&password=admin123'
```

### Ответ (успешный)

```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer",
  "expires_in": 1800
}
```

### Ошибки

| Код | Причина |
|-----|--------|
| `401` | Неверные имя пользователя или пароль |

## 📥 Эндпоинт: `POST /predict`

Принимает JSON с данными пациента и возвращает бинарное предсказание.

### Входные данные (JSON)

| Поле         | Тип    | Диапазон     | Описание |
|--------------|--------|-------------|---------|
| `Pregnancies`| `int`  | 0–20        | Количество беременностей |
| `Glucose`    | `float`| 50–200      | Уровень глюкозы (mg/dL) |
| `BMI`        | `float`| 15–50       | Индекс массы тела |
| `Age`        | `int`  | 18–90       | Возраст |

### Пример запроса

```bash
curl -X POST "http://127.0.0.1:8000/predict" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  -d '{"Pregnancies": 2, "Glucose": 140, "BMI": 35.5, "Age": 32}'
```

### Ответ (успешный)

```json
{
  "prediction": 1,
  "probability": 0.8765
}
```
> `1` — высокая вероятность диабета  
> `0` — низкая вероятность

### Ошибки

| Код | Причина |
|-----|--------|
| `401` | Неверный или отсутствующий JWT токен |
| `403` | Недостаточно прав (для некоторых эндпоинтов) |
| `422` | Некорректные данные (например, `Glucose = -1`) |
| `500` | Ошибка выполнения модели |

## 🖥️ Веб-интерфейс

На главной странице (`/`) доступен современный интерфейс с:

- Формой входа и регистрации
- Поддержкой ролей (администратор видит дополнительные функции)
- Адаптивным дизайном
- Автоматическим управлением состоянием (показ/скрытие форм)
- Кнопкой выхода
- Отображением админ-метрик для администраторов

---

## 📊 Модель

- Формат: **ONNX**
- Путь: `model/diabetes_model.onnx`
- Вход: `[Pregnancies, Glucose, BMI, Age]` (1x4)
- Выход: вероятность диабета → порог 0.5
